## Data Flow

### Project Generation Flow

```
1. User submits form
   вҶ“
2. Frontend validates input
   вҶ“
3. POST /api/projects/generate
   вҶ“
4. Backend validates user subscription (daily limit check)
   вҶ“
5. Backend constructs OpenAI prompt
   вҶ“
6. Backend streams response from OpenAI
   вҶ“
7. Backend parses & validates AI response
   вҶ“
8. Backend stores ProjectGeneration record
   вҶ“
9. Frontend receives streamed results in real-time
   вҶ“
10. User can save projects to database
    вҶ“
11. Backend creates SavedProject records
```

```
1. User clicks "Upgrade to Pro"
   вҶ“

   вҶ“

   вҶ“

   вҶ“

   вҶ“

   вҶ“

   вҶ“

   вҶ“

   вҶ“

```

## Scalability Considerations

1. **Database**
   - Connection pooling with Prisma
   - Read replicas for analytics queries
   - Caching layer (Redis) for frequently accessed data

2. **API**
   - Rate limiting per user
   - Request queuing for AI generation
   - Streaming responses for large data

3. **Frontend**
   - Code splitting
   - Lazy loading of routes
   - Service worker for offline support

4. **External Services**
   - OpenAI API rate limiting
   - Stripe webhook retries
   - Error handling and circuit breakers

## Security Architecture

1. **Authentication**
   - Clerk handles OAuth and session management
   - JWT tokens with short expiry
   - Refresh token rotation

2. **Authorization**
   - Role-based access control (RBAC)
   - Middleware checks on all protected routes
   - Database-level permission checks

3. **Data Protection**
   - HTTPS/TLS for all communications
   - Environment variables for secrets
   - Encrypted database connections

4. **API Security**
   - Rate limiting: 100 req/min per user
   - Input validation with Zod
   - CORS properly configured
   - CSRF tokens in forms

5. **Monitoring**
   - Sentry for error tracking
   - PostHog for analytics
   - Custom logging for security events
