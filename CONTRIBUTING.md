# Contributing to BuildNext

We love your input! We want to make contributing to BuildNext as easy and transparent as possible.

## Development Process

1. Fork the repo and create your branch from `main`
2. If you've added code that should be tested, add tests
3. Ensure your code passes linting and type checking
4. Make sure your code follows the style guide
5. Issue that pull request!

## Pull Request Process

1. Update the README.md with details of changes to the interface
2. Update the CHANGELOG.md with notes on your changes
3. Increase version numbers in package.json if necessary
4. Ensure all tests pass
5. Request review from maintainers

## Style Guide

### JavaScript/TypeScript
- Use ESLint configuration
- Use Prettier for formatting
- Use meaningful variable names
- Add JSDoc comments for functions

### React
- Use functional components
- Use custom hooks when appropriate
- Keep components small and focused
- Use TypeScript types

### Git Commits
- Use conventional commits: `feat:`, `fix:`, `docs:`, `style:`, `refactor:`, `test:`, `chore:`
- Write clear, concise commit messages
- One feature/fix per commit when possible

## Testing

- Write unit tests for utilities
- Write integration tests for APIs
- Write E2E tests for critical user flows
- Aim for >80% code coverage

## Getting Help

- Check existing issues and discussions
- Open a new issue if you find a bug
- Start a discussion for feature requests
- Join our Discord community

## Code of Conduct

- Be respectful and inclusive
- No harassment or discrimination
- Focus on constructive feedback
- Help each other learn and grow

Thank you for contributing to BuildNext! 🚀
